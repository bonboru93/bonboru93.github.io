tbf限速下发送速度对重传率的影响

2019-09-26

测试目的

在路由器上使用 tbf 进行限速，在服务器上调整不同的发送速度，观察对重传率的影响。

测试过程

利用三台虚拟机搭建测试环境，一台为服务器，一台为客户端，通过一台路由器连接。

使用4系内核，cubic算法。

首先在内核中添加注入点，在连接结束时输出重传率。

void middle_on_destroy_sock(struct sock* sk)
{
    struct tcp_sock *tp = tcp_sk(sk);
    if (tp->data_segs_out)
        printk("data_segs_out = %u, total_retrans = %u, retrans_rate = %u\n",
                tp->data_segs_out, tp->total_retrans, tp->total_retrans * 100 / tp->data_segs_out);
}

在路由器上使用netem设定特定重传率值后打流，观察模块输出的重传率值是否相符，修改特定数值重复测试。

实测使用【total_retrans / data_segs_out】计算重传率是准确的。

正式实验

然后正式开始实验，在路由器上通过tbf限定带宽为1mbit，也即最大发送速度128KB/S。

使用tinyhttp作为服务器，在发送时通过sleep和send的bytes数目来限制发送速度，例如限制发送速度为200KB/S：

修改发送速度，在客户端上通过ab打流，观察重传率的变化，5次取均值：

发送速度(KB/s) 重传率%
100 0
128 0
140 22
160 28
180 31
200 30
不限制 30

可以看到如果能通过上层应用将发送速度限定在tbf规定的带宽下，那么就不会有重传。超过限制带宽，重传率就会逐渐上涨，最后在本次的测试环境下最大为30%。
