DNS分流代理

2020-05-14

需求

公司内网DNS的上游不明，有些海外域名存在污染，因此想改用公共DNS。

但公司内网域名(如：xxx.company.com，company-int.com)等，公共DNS里显然是没有的。

所以萌生了搭一个DNS分流代理的想法，内网域名转内网DNS，其他域名转公共DNS。

解法

先解析DNS报文：

  1. 获取ID

    头16位就是ID。
  2. 获取请求的域名

    但这个Question Name并不直接是字符串，而是分级的，以点号作分隔，先是一个8位的长度，然后再是字符串，最后以\0结尾：

    但我们只要粗略的找这其中有没有company或company-int等关键词就可以做分流了，不需要精细解码出各个字段。

思路1

  • -监听53端口，接受DNS请求
  • -读取请求的域名
  • -根据域名进行代理选择，内网域名转内网DNS，其他域名转公共DNS
  • -新建UDP socket转发请求
  • -收到回包后转回

代码：

const dgram = require('dgram')
const server = dgram.createSocket('udp4')
 
 
server.on('message', (msg, rinfo) => {
    const queryDomain = msg.toString(undefined, 12, msg.indexOf('\0', 12))
    const client = dgram.createSocket('udp4')
    client.on('message', msg => {
        server.send(msg, rinfo.port, rinfo.address)
        client.close()
    })
    if (queryDomain.indexOf('company') >= 0)
        client.send(msg, 53, '172.22.1.1')
    else
        client.send(msg, 53, '114.114.114.114')
})
 
server.bind(53, '127.0.0.1')

这个做法非常简单，但有个问题是代理部在本地，这样每个请求都会占两个端口。

思路2

  • -监听53端口，接受DNS请求
  • -由于DNS走UDP，没有连接，所以报文里应该会需要一个ID来充当session，读出这个ID
  • -将ID和客户端的对应关系记下来，回包时用。这里有一个问题是如果两个端口发出的DNS请求恰好使用了相同的ID，就不能work了，但这种情况发生的概率不高。一定要解决的话可以把客户端和ID并在一起再做一次哈希，用这个哈希值代替原本的ID。
  • -读取请求的域名
  • -根据域名进行代理选择，内网域名转内网DNS，其他域名转公共DNS
  • -收到回包后根据ID查表得到客户端，进行转发

代码：

const dgram = require('dgram')
const server = dgram.createSocket('udp4')
const client = dgram.createSocket('udp4')
 
 
const id2port = {}
 
client.on('message', msg => {
    const queryID = msg.readUInt16BE(0)
    const port = id2port[queryID]
    if (port) {
        server.send(msg, port, '127.0.0.1')
        delete id2port[queryID]
    }
})
 
server.on('message', (msg, rinfo) => {
    const queryID = msg.readUInt16BE(0)
    id2port[queryID] = rinfo.port
    const queryDomain = msg.toString(undefined, 12, msg.indexOf('\0', 12))
    if (queryDomain.indexOf('company') >= 0)
        client.send(msg, 53, '172.22.1.253')
    else
        client.send(msg, 53, '114.114.114.114')
})
 
server.bind(53, '127.0.0.1')
