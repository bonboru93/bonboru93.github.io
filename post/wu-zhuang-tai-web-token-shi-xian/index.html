无状态web token实现

2019-11-28

需求

要实现microservice甚至serverless，就要把业务线拆解成小任务，如果用传统的session方式当user token，在各个小任务间来回同步会非常麻烦，统一管理session、处理session保持都需要性能开销。

思来想去，只能是把token保存在client侧，各任务独立验证。这就需要一种安全的在client侧保存token的加密方法。

解法

首先找到一个json web token的东西。

想法非常简单，第1段是加密方式，第2段是内容，第3段是签名。

但是jwt的签名采用的并不是严谨的非对称加密的签名，而是通过在hash函数中加自定义密文（加盐，salt，secret）的方式，加密方式和内容只是简单经过base64编码而已。

这种把摘要函数直接作为签名的方式安全性比较低，加密方式也允许client进行更改，看着着实不太踏实。

改进

想法也很简单，就是非对称加密那一套，把jwt中的摘要签名改为真正的非对称加密签名即可。python实现：

keygen.py

from ecdsa import SigningKey
 
 
sk = SigningKey.generate()
vk = sk.get_verifying_key()
with open("private.pem", "wb") as f:
    f.write(sk.to_pem())
with open("public.pem", "wb") as f:
    f.write(vk.to_pem())

server.py

from flask import Flask
from flask import request
from flask import make_response
app = Flask(__name__)
 
from ecdsa import SigningKey
with open('private.pem','rb') as f:
    priv_key = SigningKey.from_pem(f.read())
 
@app.route("/login", methods=['POST'])
def login():
    uid = request.form['uid']
    # TODO: verify uname and passwd
    expire = str(int(time.time() + 10))
    sign = priv_key.sign((uid + str(expire)).encode('ascii')).hex()
    res = make_response()
    res.set_cookie('uid', uid)
    res.set_cookie('expire', expire)
    res.set_cookie('sign', sign)
    return res
 
from ecdsa import VerifyingKey
with open('public.pem','rb') as f:
    pub_key = VerifyingKey.from_pem(f.read())
 
@app.route('/info')
def info():
    try:
        uid = request.cookies.get('uid')
        expire = request.cookies.get('expire')
        assert int(expire) > time.time()
        sign = bytes.fromhex(request.cookies.get('sign'))
        pub_key.verify(sign, (uid + str(expire)).encode('ascii'))
        return 'OK'
    except:
        return 'FAIL'

其实还可以只向client写sign，解密后用ECC做校验。
